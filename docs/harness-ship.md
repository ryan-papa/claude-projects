# Harness: 산출물 보고 + 배포

## 산출물 보고 → 사용자 승인

QA + 코드리뷰(또는 콘텐츠 검수) 통과 후 사용자에게 결과 보고:

**보고 포맷:**
- QA 결과 요약 (통과/실패 항목)
- 코드리뷰 점수 (8항목 또는 빌드 검증)
- 변경 파일 목록 + 줄 수 변화
- "승인 시 커밋→PR→배포를 진행합니다" 안내

**⛔ 사용자 승인 없이 커밋/PR/배포 진행 금지**

## 커밋 → PR → 배포

승인 후 `/rp-ship` 자동 진입:

```
git add + commit (변경 파일만)
  ↓
git push -u origin [branch]
  ↓
gh pr create (제목 + 변경 요약 + 테스트 계획)
  ↓
CI 통과 대기 → gh pr merge --merge --auto
  ↓
배포 워크플로우 완료 확인 → 라이브 URL 안내
```

## CI 규칙

| 규칙 | 내용 |
|------|------|
| CI 없을 때 | PR 생성 시점에 "CI가 없습니다. 추가할까요?" 추천 |
| 추가 여부 | 사용자 선택 시에만 추가 |
| 강제 머지 | `--admin` 플래그 사용 **금지** |
| **머지 조건** | **CI 통과 전 머지 금지** (예외 없음) |
| CI 없이 머지 | 사용자에게 수동 머지 안내 |

## 머지 후 검증

- `git log [target-branch] --oneline -3`으로 커밋 반영 확인
- 배포 워크플로우 상태 확인: `gh run list --limit 1`
- 배포 완료 시 라이브 URL 사용자에게 전달
- **배포 완료 메시지 직후 `/rp-retro` 자동 진입** (생략 불가)
